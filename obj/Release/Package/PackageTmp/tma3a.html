Part1

Original Description and requirements of the project:

The original description and requirements of part1 is to design a web application that uses a persistent cookie to track how many times a client computer has visited the page.
And at the mean time, the web should also show the time zone of where the client located and its IP address

Interpretation, Analysis and Design

Require 3 label, one to display the cookie count, one for IP address and one for the timzone

For the cookie: If the cookie do not exist, create a cookie and add to response; If the cookie exists, load the count from cookie and add one, then delete old cookie and create a new cookie and store the value and add to response

For the IP address: retrieve a list of IP address associated with the current lost and find the IP that used for inter network

For the timezone: use TimeZoneInfo class and load the ID for local time zone.

Part2

Original Description and requirements of the project:

The requirement for part2 is to develope a web application using ASP.NET in C# that runs a slideshow of a list of pictures.

  • Information about the pic must be stored either in a database or in a text file. The information should contain the name, locatoin of each picture
  • The caption should be shown under each picture
  • A button to start/stop the show
  • Control to toggle the show between random and sequential
  • buttons manually turn the show backward or forward
  • At least 20 pictures

Interpretation, Analysis and Design

Require 1 image to display the image, 1 label to display the caption, 1 script manager, 1 timer, 4 fieldsets:

  • The first fieldset contain a button that will control the start/stop action
  • The second fieldset contains control forward/backward movement for the show
  • The third fieldset contain control toggle for random/sequential movement for the show
  • The last fieldset contain two button, manually for next/previous image

The name, location and caption for the iamges will be stored inside picArray.txt. The program will read the file, and retrieve the url, caption based on index of the current image.

Since the web page reload after button click or image refresh, thus need to use ViewState to store values that need to be retained during postback There will be 4 fields need to stop inside ViewSate:

  • current: current index of the image
  • startStop: current state for start/stop
  • movement: current state for forward/backward
  • sequence: current state for random/sequential
Every time a toggle or button was clicked, it will update the value inside ViewSate. Thus the next image will change correspond to the 4 values inside ViewState
