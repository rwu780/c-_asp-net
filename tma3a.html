Part1

Original Description and requirements of the project:

The original description and requirements of part1 is to design a web application that uses a persistent cookie to track how many times a client computer has visited the page.
And at the mean time, the web should also show the time zone of where the client located and its IP address

Interpretation, Analysis and Design

Require 3 label, one to display the cookie count, one for IP address and one for the timzone

For the cookie: If the cookie do not exist, create a cookie and add to response; If the cookie exists, load the count from cookie and add one, then delete old cookie and create a new cookie and store the value and add to response

For the IP address: retrieve a list of IP address associated with the current lost and find the IP that used for inter network

For the timezone: use TimeZoneInfo class and load the ID for local time zone.

Part2

Original Description and requirements of the project:

The requirement for part2 is to develope a web application using ASP.NET in C# that runs a slideshow of a list of pictures.

  • Information about the pic must be stored either in a database or in a text file. The information should contain the name, locatoin of each picture
  • The caption should be shown under each picture
  • A button to start/stop the show
  • Control to toggle the show between random and sequential
  • buttons manually turn the show backward or forward
  • At least 20 pictures

Interpretation, Analysis and Design

Require 1 image to display the image, 1 label to display the caption, 1 script manager, 1 timer, 4 fieldsets:

  • The first fieldset contain a button that will control the start/stop action
  • The second fieldset contains control forward/backward movement for the show
  • The third fieldset contain control toggle for random/sequential movement for the show
  • The last fieldset contain two button, manually for next/previous image

The name, location and caption for the iamges will be stored inside picArray.txt. The program will read the file, and retrieve the url, caption based on index of the current image.

Since the web page reload after button click or image refresh, thus need to use ViewState to store values that need to be retained during postback There will be 4 fields need to stop inside ViewSate:

  • current: current index of the image
  • startStop: current state for start/stop
  • movement: current state for forward/backward
  • sequence: current state for random/sequential
Every time a toggle or button was clicked, it will update the value inside ViewSate. Thus the next image will change correspond to the 4 values inside ViewState

Documentation:

mai.aspx: main page for part2, contain the slideshow

picArray.txt: contain the caption and the source of different pics

pic folder: contiin all the picture used for the application

Part3

Original Description and requirements of the project:

The requirment for part3 is to develop an online computer store in which customers can select a computer from a list and customize it by replacing some components in its standard configuration with other that may be better or cheaper. Customer can then place an order with the online store.

  • There should be a list of computers, lists of different parts or componets(no less than five different components including at least RAM, hard drive , CPU, display, OS, and soundcard) of computers that can be reviewed by customers.
  • A customer can select a computer with default/standard configuration and pricing, and then customize it by changing its configuration
  • Whenever a change has been made by the customer, the price total needs to be dynamically changed and displayed to the customer
  • After the customer has hit the buy, a summary of selections, including all details of the customized computer and its pricing , should be displayed in the browser window.
  • At this stage, you don't need to save orders placed by customers in the database.

Interpretation, Analysis and Design

Require three aspx page for this part

  • The first page contain a welcome page to the user
  • The second page contain is the order page
  • The third page contain a contact/feedback form

The first page will contain all the links to the other pages, it will contain greeting to users and a brief description of the application

The second page contain all the computer and component information. Since all value in part3 is hard code. When the user submitting the order, it just added up all the value and display the final price

The third page contain a feedback form. It will contain name, email, phone number, subject and comment field. All field will be validate by the validator before submit. After submit, it should have a confirm message for the user.

Findings: from online stores,

  1. User able to explore product without signing in
  2. User only need to sign in only when he/she placed the order or want to view his own order
  3. Price dynamically update when user add/delete an item
  4. User able to see the reference number of his/her order
  5. Items were listed under their own departments and their own categories, thus user can navigate the page and find what they want
Apply to own app
  1. User should able to view the product without sign in
  2. User able to sign in to view his/her orders
  3. If the user did not sign when he/she place an order, the program will ask the user to sign in or register
  4. User able to see the reference number of his/her order
